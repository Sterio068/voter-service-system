# 全系統全功能測試計畫

本文件用於登入實機系統後逐項巡檢功能、UI/UX、權限、資料一致性與維運流程。目標不是只確認「頁面打得開」，而是確認服務處日常工作流能穩定完成，且敏感資料操作有足夠保護與稽核。

## 0. 測試前準備

測試帳號：
- `admin`：完整巡檢、設定、備份、匯出、帳號管理。
- `supervisor`：業務全功能與報表巡檢，確認不可進入帳號維護。
- `assistant`：日常 CRUD 巡檢，確認後台與完整匯出受限。
- `volunteer`：唯讀、電話拜訪與低敏資料顯示巡檢。

測試資料建議：
- 至少 5 筆選民，含手機、地址、標籤、生日、支持度、介紹人。
- 至少 3 筆陳情，涵蓋待處理、處理中、已結案與高急迫。
- 至少 2 筆公文，分別為收文與發文。
- 至少 3 筆行程，包含一般、跨日、公祭或禮儀類型。
- 至少 1 個團體，含 2 位成員。
- 至少 1 筆活動、問卷、通知、提案、禮儀記錄、廠商與支出。

缺陷記錄格式：
- 模組：
- 角色：
- 步驟：
- 預期結果：
- 實際結果：
- 嚴重度：P0 阻斷 / P1 高風險 / P2 可繞過 / P3 體驗改善
- 截圖或錯誤訊息：

## 1. 共用 UI/UX 巡檢

所有主要頁面都要檢查：
- 頁面有一致的 `PageScaffold` 標題、eyebrow、描述與 action bar。
- 列表頁篩選使用 `WorkspaceToolbar`，搜尋、篩選、統計與清除篩選位置一致。
- 空資料與查無資料使用 `EmptyState`，文案能引導下一步。
- Drawer 或 Modal 表單底部使用一致的取消/儲存操作。
- 長表單以 `FormSection` 分段，使用者能理解每一段的目的。
- 手機寬度下頁面不橫向溢出，主要操作按鈕仍可點擊。
- Dark mode 下卡片、表單分段、表格、空狀態對比足夠。
- 全站搜尋 `Ctrl+K` 可開啟並跳轉結果。
- 快捷鍵 `?` 可開啟說明。

## 3. 橫向測試場景

資料生命週期：
- 新增選民後建立陳情、待辦、行程、團體成員，確認詳情頁與報表同步。
- 軟刪選民後，列表與匯出不應出現；audit 仍保留。
- 合併選民後，陳情、聯絡、待辦、活動、問卷關聯都指向主選民。

個資與匯出：
- 一般選民匯出固定遮罩身分證、手機、生日、地址、Email、LINE。
- 完整匯出只有 admin 可用，理由必填，audit detail 可查。
- 列印名冊與地址標籤需確認欄位選擇與遮罩政策。

權限：
- 逐一切換 admin、supervisor、assistant、volunteer，確認導航、API 403、按鈕可見性一致。
- 志工不得看到帳號、設定、完整匯出、刪除與批次修改。

可靠性：
- 建立備份後驗證 metadata 顯示已簽章。
- 還原前必須有明確警示與完整性檢查結果。
- 還原後重新啟動時，若存在 pending restore，正式版應在啟動後顯示「還原完成」，且資料應切換到備份內容。
- 自動備份失敗需在設定頁顯示最後錯誤。
- fresh install 時首次執行精靈必須完成管理員密碼修改後才能結束；不得再出現可略過或失敗後直接完成的路徑。
- 正式版應由 `http://127.0.0.1:8080` 提供 built assets；首頁、登入、主要列表頁不可再依賴 `file://` 模式假設。

整合：
- Google Calendar 同步失敗不可阻擋行程新增。
- LINE webhook 或通知設定未完成時，不可影響核心 CRUD。
- AI provider 未設定或失敗時，AI 按鈕需顯示錯誤但不可清空表單。
- `client-errors` 僅允許已登入 session 上報；登入前頁面錯誤不應再匿名寫入資料庫。
- 單筆匿名化需確認 `anonymize` 與 `full` 兩種模式都有明確差異，且 tags/topics/relations/engagement/contacts 等關聯同步清理。

## 4. 自動化測試對照

目前可重跑指令：
- `npm run typecheck`
- `npm test`
- `npm run test:e2e -- --project=chromium`
- `npm run verify`
- `git diff --check`

已覆蓋：
- API integration：auth、permissions、first-run guard、client-errors auth、voters、voter anonymize/full mode、petitions、import/export、attachments、schedules、consultations、backup/restore、data retention、secrets round-trip。
- E2E smoke：登入、Dashboard deep-link、主要模組 shell、UI 新增選民、UI 新增陳情、備份建立、完整匯出理由與下載。
- E2E navigation：admin 可開啟所有主要路由，且不出現 ErrorBoundary 崩潰頁。
- E2E role access：assistant、supervisor、volunteer 的導覽、受限路由、快捷鍵、Dashboard 入口、列印頁、禮儀/收支 read-only 管控與頁內 create/edit/delete 按鈕不會暴露超出角色矩陣的功能。
- 正式版隔離驗收：fresh install wizard、pending restore 重啟套用、`backup_path` 初始值，以及 12,000 選民 / 1,500 陳情 / 300 行程的 packaged load smoke。
- 正式版本機 smoke：`/Applications/選民服務系統.app` 啟動、`/api/health` 正常、admin 登入、選民列表載入、匿名 `client-errors` 被拒絕、登入後可上報，以及 `mode=full` 匿名化實際清空延伸欄位。
