# Data Protection Runbook

## 基本原則

- 選民資料屬敏感個資，預設只使用遮罩匯出。
- 完整個資匯出只允許 admin，且必須填寫匯出理由。
- 備份、Excel、截圖、列印文件都視為個資載體。

## 匯出控管

1. 一般名冊、統計、查詢交付：使用「匯出 Excel（遮罩）」。
2. 郵寄、正式通知、法定用途：由 admin 使用「完整匯出」並填寫理由。
3. 匯出後檔案應存放於加密磁碟，完成用途後刪除。
4. 每週檢查 audit logs 中 `voter_export` 記錄。

## 資料品質掃描

1. 進入「系統設定 → 資料品質掃描」。
2. 檢查重複手機、重複身分證、孤兒附件、遺失附件檔案。
3. 高風險項目先人工確認，再安排修復。

## 資料保留

1. 先確認最新備份顯示「已簽章」，並執行備份驗證。
2. 在「系統設定 → 資料保留政策」設定保留天數並儲存。
3. 在「資料保留預覽」確認待封存、待刪除與待去識別筆數。
4. 確認無誤後輸入 `RUN_RETENTION` 執行。
5. 執行後抽查 audit archive、client_errors 筆數與已停用選民去識別狀態。

## 外洩疑慮處理

1. 立即停止相關帳號與匯出操作。
2. 保存 audit logs、備份 metadata、相關 Excel 檔案資訊。
3. 確認外洩範圍：匯出時間、匯出者、篩選條件、筆數、是否完整個資。
4. 輪替 `VOTER_SERVICE_SETTINGS_KEY` 需先評估既有 secrets 是否可解密；不可直接替換。
5. 依組織個資事件通報流程處理。

## 最低部署要求

- 啟用 OS full-disk encryption。
- 使用受控帳號與最小權限。
- 區網多人使用時應加 HTTPS 或 VPN。
- 備份位置必須在加密磁碟或受控 NAS。
